# Debugging Lessons

- 2026-05-08: Source files were originally under `src/` while imports expected `option_data_manager`; standard package structure was required before API/WebUI could run.
- 2026-05-11: Public server realtime Quote worker can exit if TQSDK `get_quote_list` times out on a large exchange/product batch. Treat batch Quote subscription like Kline batch subscription: fall back to single-symbol `get_quote`, skip only symbols that still fail, and keep the long-lived worker alive. For small test servers, keep `quote_stream.workers=1`, `quote_stream.quote_shard_size` around 50, and `quote_stream.kline_batch_size=1` until live evidence supports higher values.
- 2026-05-13: A realtime worker can show Quote subscriptions complete while T型报价 stays static if visible Quote writes wait behind the long Kline setup phase. Keep Quote writes alive during Kline setup, use batch SQLite transactions for startup snapshots, and do not enqueue metrics dirty work from startup snapshot seeding. Do not make WebUI/T型报价 requests responsible for refreshing data; realtime data must advance from backend workers even when no browser is open.
- 2026-05-14: Do not treat TQSDK `trading_time` as the sole source for current session state. Live payloads can have `trading_time.night=[]` while the underlying Quote `datetime/source_datetime` is actively advancing during a night session. For UI display and moneyness session gating, fresh Quote source time is authoritative live evidence; theoretical `trading_time` ranges are only fallback data when Quote time is stale or absent.
- 2026-05-15: In realtime WebUI paths, do not collapse all missing Quote times into one state. Before the row is fully subscribed, display `待行情`; after the row is `已订阅`, a missing underlying Quote time is closed-session evidence and should display `休市`.
- 2026-05-15: T型报价 stale-data masking must be Quote-subscription-aware, not purely timestamp-based and not gated by Kline completion. Before a chain's Quote objects are fully subscribed, stale cached values should render as `--`; after `quote_subscribed == quote_total`, closed-session cached Quote/IV/Greeks are the best current slice and should remain visible while Kline continues catching up.
- 2026-05-15: Broad Quote subscription and moneyness-filtered Kline subscription are different display contracts. T型报价 may use broad Quote completion to allow current cached values, but it must still mask option legs outside the active moneyness Kline filter; otherwise historical ITM/ATM cache leaks onto an OTM-only screen and looks like data that was intentionally subscribed.
- 2026-05-15: Moneyness masking must be recomputed on every T型报价 refresh from the latest underlying Quote price. Do not cache per-row display scope across requests; price moves can make a previously hidden ITM leg become OTM and visible, or make a previously visible OTM leg become ITM/ATM and hidden.
- 2026-05-15: During Kline setup, forced Quote snapshots must not write stale TQSDK quote refs after `wait_update` times out. A fresh `received_at` on an old `source_datetime` made rows look current while the underlying market price was stale. Use the normal wait deadline and only write the forced snapshot when `wait_update` succeeds; longer term, split Quote and Kline maintenance into separate workers so Kline object setup cannot starve Quote freshness.
- 2026-05-15: T型报价 coverage metrics must use the same display/subscription scope as the table body. When only OTM legs are subscribed, IV/Greeks/Kline coverage denominators should exclude hidden ITM/ATM legs; also treat numeric zero IV/Greeks values as present data, not as missing values.
- 2026-05-15: Do not rely on a hardcoded frontend product-code map as the primary Chinese product-name source. TQSDK Quote payloads include `instrument_name` such as `铝合金2606` and `氧化铝2606`; derive the display product from that when available, and keep static mappings as fallback only.
